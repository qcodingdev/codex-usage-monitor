# Security policy

## Scope

Codex Usage Monitor is a local read-only companion for Codex. It should never require a user to paste a ChatGPT or Codex token into the project.

## Reporting a vulnerability

Please do not open a public Issue for credentials, token exposure, or a reproducible security vulnerability. Contact the maintainer through the private contact method listed on the GitHub profile and include the affected version, platform, reproduction steps, and a minimal sanitized example.

## Data-handling promise

The project does not intentionally upload usage data or authentication data. Reports and pull requests must remove account identifiers, local paths, logs, and screenshots that contain private information.
