# Changelog

## 0.1.4 - 2026-07-24

- 修复 app-server 退出后标准输出 EOF 回调可能重复触发、导致 CPU 占用异常的问题。

## 0.1.3 - 2026-07-24

- 每次用量读取完成后，app-server 会在 10 秒空闲期后自动退出，释放常驻内存。
- 悬浮窗最小化后，后台刷新从 60 秒降为 180 秒；再次打开会恢复 60 秒。

## 0.1.2 - 2026-07-16

- 修复首次打开时可能先显示“暂不可用”、需要手动刷新的问题。
- 初始化或读取失败后会彻底清理失效的 app-server 连接。
- 启动阶段增加自动退避重试，并防止旧进程退出事件干扰新连接。
- 插件的一次性用量查询也会在瞬时失败时自动重试。

## 0.1.1 - 2026-07-16

- 增加双击安装程序，将应用安装到用户“应用程序”目录。
- 增加 `codex-usage-monitor` CLI，可启动、查看状态和定位应用。
- “退出”改为“完全退出”，退出前提示重新打开方式。
- 插件优先启动已安装应用，并保留内置应用回退。
- macOS 应用改为 Universal 构建，同时支持 Intel 与 Apple Silicon。

## 0.1.0 - 2026-07-13

- 首个公开 macOS MVP。
- 支持 5 小时/7 天窗口、重置时间、套餐、Credits 和 Token 活动。
- 支持 Codex Desktop 内置 CLI 和独立 Codex CLI 的自动发现。
- 增加菜单栏、悬浮面板、最小化和退出控制。
- 增加插件技能、源码构建脚本和本地只读安全边界。
